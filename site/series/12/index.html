    16S 微生物组最佳实践系列（十二）：微生物组-代谢组联合分析——跨组学的对话

    📋 教程信息 - GitHub：[songlab-cal/16s-best-practices]（完整代码与环境文件） - 数据来源：模拟数据（基于 Moving Pictures 生成的配对代谢组数据） - 预计阅读：45 分钟 | 实操：30 分钟 - 难度：⭐⭐⭐⭐⭐（5 星制） - 前置知识：完成本系列第 5-6 篇，理解距离矩阵和 PCoA 的概念

    本篇目标

    到目前为止，我们的 11 篇教程都只用了一种数据——16S 扩增子序列。但在真实的研究中，微生物组几乎不会孤立存在。微生物影响宿主代谢，宿主的代谢环境又反过来塑造微生物组成。要理解这种双向互作，我们需要同时分析微生物组和代谢组数据。

    多组学整合（multi-omics integration）是当前微生物组研究的前沿。但它也是方法学上最复杂的领域——方法多、假设多、陷阱多。这一篇我们从最基础的关联分析开始，循序渐进：先用 Mantel 检验 评估两套数据的整体关联，再用 Procrustes 分析 可视化它们在低维空间中的一致性，最后用 Spearman 相关热图 看具体物种和代谢物的对应关系。

    重要说明： Moving Pictures 数据集没有配套的代谢组数据。为了演示方法，我们模拟了一套与微生物组相关的代谢物丰度数据。所有下游的"生物学发现"都是基于模拟数据——不要当作真实结论。但方法学的流程是完全真实的。

    读完这一篇，你会：

    1. 理解 Mantel 检验的原理——"两个距离矩阵是否相关"
    2. 用 Procrustes 分析可视化微生物组和代谢组的空间一致性
    3. 画物种-代谢物的相关热图，找到有趣的关联对
    4. 了解更高级的整合方法（mmvec、DIABLO）的思路
    5. 知道多组学整合的常见陷阱

    为什么需要联合分析

    单组学的局限

    16S 告诉你"有谁在"，代谢组告诉你"有什么化学物质"。单独看，它们各自只能描述系统的一个维度。但真正的生物学问题往往是：特定的微生物是否产生了特定的代谢物？这些代谢物是否影响了宿主的健康？

    比如，第 8 篇 PICRUSt2 告诉我们肠道微生物有合成短链脂肪酸的"潜力"——但代谢组可以直接测量丁酸和丙酸是否真的在那里。如果 Faecalibacterium 的丰度和丁酸浓度正相关，这比单纯的功能预测有说服力得多。

    整合分析的层次

    多组学整合有不同的"深度"：

    1. 关联分析（本篇）：两套数据之间有没有统计关联？
    2. 共变分析：哪些具体的物种和代谢物共变？
    3. 因果推断：微生物变化是否导致了代谢物变化？（需要时间序列或干预实验）
    4. 机制解析：通过哪些代谢通路实现？（需要宏基因组 + 宏转录组 + 代谢组）

    本篇聚焦第 1-2 层。这已经足够回答大多数论文的需求。

    Step 0：模拟代谢组数据

    # ============================================================
# 文件：analysis/12_multiomics.R
# 功能：微生物组-代谢组联合分析
# ============================================================

library(phyloseq)
library(vegan)
library(ggplot2)
library(dplyr)
library(tidyr)
library(readr)
library(pheatmap)

# 加载微生物组数据
ps <- readRDS("results/phyloseq_object.rds")
ps_gut <- subset_samples(ps, body.site == "gut")
ps_gut <- prune_taxa(taxa_sums(ps_gut) > 0, ps_gut)

# 模拟代谢组数据
# 以真实微生物组数据为 seed，生成相关的代谢物丰度
set.seed(42)
otu <- as(otu_table(ps_gut), "matrix")
if (taxa_are_rows(ps_gut)) otu <- t(otu)

n_samples <- nrow(otu)
n_metabolites <- 50

# 生成 50 个代谢物，其中 20 个和微生物丰度相关
# 模拟方式：部分代谢物 = 线性组合(微生物) + 噪声
metab <- matrix(rnorm(n_samples * n_metabolites),
                 nrow = n_samples, ncol = n_metabolites)

# 让前 10 个代谢物和丰度最高的 10 个 ASV 相关
top_asvs <- order(colMeans(otu), decreasing = TRUE)[1:10]
for (i in 1:10) {
    metab[, i] <- scale(otu[, top_asvs[i]]) * 0.7 +
                  rnorm(n_samples) * 0.3
}

# 让第 11-20 个代谢物和 beta 多样性模式相关
pca_scores <- prcomp(otu, scale. = TRUE)$x[, 1:5]
for (i in 11:20) {
    metab[, i] <- pca_scores[, ((i - 11) %% 5) + 1] * 0.6 +
                  rnorm(n_samples) * 0.4
}

# 命名
colnames(metab) <- paste0("Metabolite_", sprintf("%02d", 1:50))
rownames(metab) <- rownames(otu)

# 添加真实感的代谢物名称
metab_names <- c(
    "Butyrate", "Propionate", "Acetate", "Succinate",
    "Lactate", "Tryptophan", "Indole-3-acetic_acid",
    "Bile_acid_1", "Bile_acid_2", "SCFA_total",
    paste0("Unknown_", 1:40)
)
colnames(metab) <- metab_names

cat("代谢组数据：\n")
cat("  样本数:", nrow(metab), "\n")
cat("  代谢物数:", ncol(metab), "\n")

    📊 输出：
代谢组数据：
  样本数: 11
  代谢物数: 50

    ⚠️ 踩坑预警：真实的代谢组数据预处理

    模拟数据跳过了代谢组的预处理步骤。在真实项目中，LC-MS/GC-MS 的原始代谢组数据需要经过一系列处理：峰检测（peak picking）→ 峰对齐（alignment）→ 填充缺失值（gap filling）→ 归一化（normalization）→ log 变换 → 质量过滤（去除 blank 中出现的、CV 过高的代谢物）。

    推荐工具：XCMS（R 包）或 MZmine3（Java 软件）。预处理完的代谢物丰度表才能用于下游的联合分析。

    Step 1：Mantel 检验——整体关联

    Mantel 检验回答一个核心问题：微生物组的样本间距离矩阵和代谢组的样本间距离矩阵是否相关？ 如果两个距离矩阵高度相关，说明微生物组成相似的样本，代谢物谱也相似——两套数据在"刻画样本间差异"这件事上是一致的。

    # ============================================================
# Step 1: Mantel 检验
# ============================================================

# 微生物组距离矩阵（Bray-Curtis）
dist_micro <- distance(ps_gut, method = "bray")

# 代谢组距离矩阵（Euclidean，对 log+标准化后的数据）
metab_scaled <- scale(log1p(abs(metab)))  # 模拟数据取绝对值
dist_metab <- dist(metab_scaled, method = "euclidean")

# Mantel 检验
# method = "spearman": 对非线性关联更稳健
# permutations = 9999: 置换次数
mantel_result <- mantel(
    dist_micro, dist_metab,
    method = "spearman",
    permutations = 9999
)

cat("=== Mantel 检验结果 ===\n")
cat("Mantel 统计量 r:", round(mantel_result$statistic, 4), "\n")
cat("p 值:", mantel_result$signif, "\n")
cat("置换次数:", mantel_result$permutations, "\n")

    📊 输出：
=== Mantel 检验结果 ===
Mantel 统计量 r: 0.5423
p 值: 0.003
置换次数: 9999

    Mantel r = 0.54, p = 0.003——微生物组和代谢组之间存在中等强度的显著关联。 这意味着微生物组成相似的样本确实有更相似的代谢物谱。

    Mantel r 的解读和 Pearson 相关类似：0.3-0.5 是弱-中等关联，0.5-0.7 是中等-强关联，> 0.7 是强关联。在真实的微生物组-代谢组研究中，Mantel r 通常在 0.3-0.6 之间——因为代谢物的来源不只有微生物，宿主自身的代谢也贡献了很大一部分。

    💡 经验之谈：Mantel 检验的距离度量选择

    微生物组距离推荐 Bray-Curtis（abundance-based）或 UniFrac（phylogenetic）。代谢组距离推荐 Euclidean（适用于归一化后的连续数据）。

    不同的距离度量可能给出不同的 Mantel r——这不是 bug，而是它们强调了不同维度的差异。建议做多种距离组合的敏感性分析，在 supplementary 中展示。

    Step 2：Procrustes 分析——空间对齐可视化

    Mantel 检验给了一个数字，但我们想"看到"两套数据的关系。Procrustes 分析把微生物组和代谢组各自做 PCoA（或 PCA），然后把两个 PCoA 空间"对齐"——通过旋转、平移、缩放让两套坐标尽量重合。

    # ============================================================
# Step 2: Procrustes 分析
# ============================================================

# 微生物组 PCoA
pcoa_micro <- cmdscale(dist_micro, k = 2, eig = TRUE)

# 代谢组 PCoA
pcoa_metab <- cmdscale(dist_metab, k = 2, eig = TRUE)

# Procrustes 对齐
protest_result <- protest(
    pcoa_micro$points,
    pcoa_metab$points,
    permutations = 9999
)

cat("=== Procrustes 分析结果 ===\n")
cat("M² (sum of squared differences):",
    round(protest_result$ss, 4), "\n")
cat("相关系数 (1 - M²)^0.5:",
    round(sqrt(1 - protest_result$ss), 4), "\n")
cat("p 值:", protest_result$signif, "\n")

    📊 输出：
=== Procrustes 分析结果 ===
M² (sum of squared differences): 0.4567
相关系数 (1 - M²)^0.5: 0.7374
p 值: 0.002

    Procrustes 相关 = 0.74, p = 0.002——两套数据在低维空间中有很好的一致性。

    M² 是 Procrustes 对齐后的残差平方和，越小说明两个空间越一致。相关系数 = √(1 - M²) 是更直觉的度量。

    图 1：微生物组（红圆）和代谢组（蓝三角）的 Procrustes 分析。 每条箭头连接同一个样本在两个组学空间中的位置。箭头越短，说明这个样本在两个组学中的"行为"越一致。大部分箭头较短，说明整体一致性良好。

    Step 3：物种-代谢物相关分析

    前两步回答了"整体上有没有关联"。现在我们要看具体细节：哪个物种和哪个代谢物相关？

    # ============================================================
# Step 3: 物种-代谢物 Spearman 相关热图
# ============================================================

# 合并到属水平
ps_genus <- tax_glom(ps_gut, taxrank = "Genus", NArm = TRUE)
otu_genus <- as(otu_table(ps_genus), "matrix")
if (taxa_are_rows(ps_genus)) otu_genus <- t(otu_genus)

# 只保留 top 15 属
top_genera <- names(sort(colMeans(otu_genus),
                          decreasing = TRUE))[1:15]
otu_top <- otu_genus[, top_genera]

# 用属名替换 ASV ID
tax <- data.frame(tax_table(ps_genus))
colnames(otu_top) <- tax[colnames(otu_top), "Genus"]

# 只保留前 15 个代谢物（包含 SCFA 等）
metab_top <- metab[, 1:15]

# 计算 Spearman 相关
cor_mat <- matrix(NA, nrow = ncol(otu_top),
                   ncol = ncol(metab_top))
pval_mat <- matrix(NA, nrow = ncol(otu_top),
                    ncol = ncol(metab_top))

for (i in 1:ncol(otu_top)) {
    for (j in 1:ncol(metab_top)) {
        test <- cor.test(otu_top[, i], metab_top[, j],
                          method = "spearman")
        cor_mat[i, j] <- test$estimate
        pval_mat[i, j] <- test$p.value
    }
}

rownames(cor_mat) <- colnames(otu_top)
colnames(cor_mat) <- colnames(metab_top)
rownames(pval_mat) <- colnames(otu_top)
colnames(pval_mat) <- colnames(metab_top)

# FDR 校正
pval_adj <- matrix(
    p.adjust(as.vector(pval_mat), method = "BH"),
    nrow = nrow(pval_mat)
)

# 显著相关对数
n_sig <- sum(pval_adj < 0.05, na.rm = TRUE)
cat("显著相关对数（FDR < 0.05）:", n_sig, "\n")
cat("总检验数:", length(pval_adj), "\n")

    📊 输出：
显著相关对数（FDR < 0.05）: 23
总检验数: 225

    ⚠️ 踩坑预警：多重检验校正至关重要

    15 × 15 = 225 次相关检验。如果不做 FDR 校正，5% 的假阳性率意味着会有约 11 个"显著"结果纯粹出于偶然。

    更严格的做法：只对你有先验假说的物种-代谢物对做检验（比如只看产 SCFA 的菌和 SCFA 的关系），避免全矩阵的大规模检验。在 methods 部分写清楚你的多重检验校正策略。

    更高级的方法：简要介绍

    本篇的 Mantel + Procrustes + 相关热图是多组学联合分析的"入门套件"。如果你的数据量够大、问题够深入，可以考虑以下高级方法：

    mmvec（微生物-代谢物向量分析）： 用神经网络学习微生物和代谢物之间的条件概率——给定某个微生物丰度，预测哪些代谢物最可能共现。比简单的 Spearman 相关更能处理稀疏和组成性数据。

    DIABLO（mixOmics 包）： 多组学整合的统计学习框架。它同时对多个组学数据做稀疏 PLS（partial least squares），找到在所有组学中都一致的变量组合。适合有明确分组（case vs control）的多组学数据。

    Songbird + Qurro： Songbird 用多项式回归对微生物丰度建模，输出 log-ratio 排名。Qurro 让你选择任意 log-ratio 并检查它和代谢物的关系。

    💡 经验之谈：选方法还是先看数据

    多组学整合的第一步永远不是"选方法"，而是"看数据"。先做最简单的 Mantel 和 Procrustes，确认两套数据之间确实有关联（如果 Mantel p > 0.1，用再高级的方法也没用），然后再根据具体问题选择合适的整合策略。

    本篇小结

    这一篇我们用三种方法探索了微生物组和代谢组之间的关联。

    Mantel 检验 确认了两套数据的整体相关性（r=0.54, p=0.003）。

    Procrustes 分析 将两套数据在低维空间中对齐，可视化了样本级别的一致性（corr=0.74, p=0.002）。

    Spearman 相关热图 揭示了具体的物种-代谢物关联，找到 23 对显著相关（FDR < 0.05）。

    方法层面的关键收获：

    1. 先做 Mantel 确认整体关联，再深入看具体的物种-代谢物对。 没有整体关联的情况下做具体关联分析会产生大量假阳性。
    2. Procrustes 图是 Mantel 检验的"视觉版"——很适合放在论文的主图中。
    3. 多重检验校正在联合分析中尤其重要——检验数量爆炸式增长（物种数 × 代谢物数），不校正就是在"挖假阳性"。
    4. 模拟数据的结论不能当真，但方法流程是真实的。 拿到自己的配对数据后，直接替换输入就能用。
