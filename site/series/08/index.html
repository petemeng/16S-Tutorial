    16S 微生物组最佳实践系列（八）：PICRUSt2 功能预测——它们能做什么

    📋 教程信息 - GitHub：petemeng/16S-Tutorial（完整代码与环境文件） - 数据来源：Atacama soils 双端数据集（54 个样本，1,078 个 ASV；PICRUSt2 功能预测） - 预计阅读：40 分钟 | 实操：30 分钟 - 难度：⭐⭐⭐⭐（5 星制） - 前置知识：完成第 3-7 篇，results/ 下已有 table-filtered.qza、rep-seqs-filtered.qza 和 phyloseq_object.rds

    本篇目标

    上一篇我们回答了“谁变了”。但审稿时常会被继续追问：这些类群变化意味着什么功能差异？

    16S 本身并不能直接测功能基因，所以这里我们走的是 PICRUSt2 路线：用 ASV 序列在参考树上的位置，去推断群落可能携带哪些代谢通路。

    读完这一篇，你会：

    1. 理解 PICRUSt2 的基本原理和局限
    2. 从 Atacama 的 ASV 表和代表序列直接跑通 PICRUSt2
    3. 看懂 NSTI 这个最重要的质量指标
    4. 比较 Baquedano 和 Yungay 的预测通路差异
    5. 知道怎么在论文里谨慎使用“预测功能潜力”这类结果

    先说结论：PICRUSt2 是“预测”，不是“测量”

    这点必须一开始就讲清楚。

    PICRUSt2 的逻辑是：

    1. 把你的 ASV 放到参考系统发育树上
    2. 找它最近的已测序参考基因组
    3. 用系统发育邻近关系外推这个 ASV 可能带哪些基因
    4. 按样本丰度加权求和，得到整个群落的功能潜力

    所以 PICRUSt2 输出的不是 observed function，而是 predicted functional potential。

    这类结果能不能信，很大程度上取决于你的 ASV 离参考基因组到底有多近。这个距离就是 NSTI。

    环境准备

    在这台服务器上，PICRUSt2 没有装进 QIIME2 主环境，而是单独放进了一个 picrust2 conda 环境，避免依赖冲突。

    # ============================================================
# PICRUSt2 环境
# ============================================================

conda create -n picrust2 -c bioconda -c conda-forge picrust2=2.5.3 -y
conda run -n picrust2 picrust2_pipeline.py --version

    📊 输出：
PICRUSt2 version 2.5.3

    如果你已经装好这个环境，这一段可以直接跳过。

    Step 1：导出 PICRUSt2 需要的输入文件

    PICRUSt2 需要两个东西：

    1. feature table，BIOM 格式
    2. 代表序列，FASTA 格式

    在 Atacama 这套流程里，我们直接从过滤后的 table-filtered.qza 和 rep-seqs-filtered.qza 导出。

    # ============================================================
# 文件：analysis/08_picrust2.R
# 功能：运行 PICRUSt2 并整理差异通路结果
# ============================================================

source("/media/desk16/tly9658/16s-atacama-tutorial/analysis/common_16s.R")

suppressPackageStartupMessages({
  library(dplyr)
  library(ggplot2)
  library(readr)
  library(tidyr)
  library(stringr)
  library(tibble)
})

run_env_cmd <- function(env_name, args) {
  status <- system2("conda", c("run", "-n", env_name, args))
  if (!identical(status, 0L)) {
    stop("Command failed in env ", env_name, ": ", paste(args, collapse = " "))
  }
}

picrust_input <- file.path(ATACAMA_ROOT, "results", "picrust2_input")
picrust_out <- file.path(ATACAMA_ROOT, "results", "picrust2_out")
unlink(picrust_input, recursive = TRUE, force = TRUE)
unlink(picrust_out, recursive = TRUE, force = TRUE)
dir.create(picrust_input, recursive = TRUE, showWarnings = FALSE)

run_env_cmd("qiime2-amplicon-2024.5", c(
  "qiime", "tools", "export",
  "--input-path", file.path(ATACAMA_ROOT, "results", "table-filtered.qza"),
  "--output-path", picrust_input
))
run_env_cmd("qiime2-amplicon-2024.5", c(
  "qiime", "tools", "export",
  "--input-path", file.path(ATACAMA_ROOT, "results", "rep-seqs-filtered.qza"),
  "--output-path", picrust_input
))

cat("PICRUSt2 输入文件：\n")
print(list.files(picrust_input))

    Step 3：先看 NSTI，判断这批结果值不值得继续解读

    这一步不能省。NSTI 太差，后面所有“功能差异”都应该只当探索性结果。

    # ============================================================
# Step 3: NSTI 统计
# ============================================================

nsti_df <- read_tsv(
  gzfile(file.path(picrust_out, "marker_predicted_and_nsti.tsv.gz")),
  show_col_types = FALSE
)
nsti_col <- tail(colnames(nsti_df), 1)

nsti_stats <- tibble(
  feature_count = nrow(nsti_df),
  nsti_median = median(nsti_df[[nsti_col]], na.rm = TRUE),
  nsti_mean = mean(nsti_df[[nsti_col]], na.rm = TRUE),
  nsti_min = min(nsti_df[[nsti_col]], na.rm = TRUE),
  nsti_max = max(nsti_df[[nsti_col]], na.rm = TRUE),
  pct_lt_0_03 = mean(nsti_df[[nsti_col]] < 0.03, na.rm = TRUE) * 100,
  pct_lt_0_15 = mean(nsti_df[[nsti_col]] < 0.15, na.rm = TRUE) * 100
)

write_tsv(nsti_stats, file.path(ATACAMA_ROOT, "results", "picrust2_nsti_summary.tsv"))

cat("NSTI 统计：\n")
print(nsti_stats)

    📊 输出：
NSTI 统计：
  feature_count nsti_median nsti_mean nsti_min nsti_max pct_lt_0_03 pct_lt_0_15
1         1078       0.2820    0.3618   0.0001   2.7479        8.35        24.21

    这个结果的信号很明确：

    中位数 0.282 明显不低。 对土壤/荒漠环境样本来说，这不算罕见，但也意味着这批功能预测不能像人类肠道那样放心大胆地解读。

    只有 24.2% 的 ASV NSTI < 0.15。 说明参考基因组覆盖并不理想，很多 ASV 都是在相对较远的参考上外推出来的。

    所以这一章的正确定位应该是：探索性功能趋势，不是定论。

save_plot_dual(p_nsti, "ch08_nsti_histogram", width = 7.5, height = 5)

    Step 4：比较不同 transect 的预测通路差异

    虽然 NSTI 不算理想，我们仍然可以把它当成“候选功能趋势”来探索。这里我们直接比较 MetaCyc 通路在 Baquedano 和 Yungay 之间的差异。

    # ============================================================
# Step 4: 通路差异分析
# ============================================================

pathway_path <- file.path(picrust_out, "pathways_out", "path_abun_unstrat.tsv.gz")
pathway_df <- read_tsv(gzfile(pathway_path), show_col_types = FALSE)

sample_meta <- readRDS(file.path(ATACAMA_ROOT, "results", "phyloseq_object.rds")) %>%
  sample_data() %>%
  data.frame(check.names = FALSE) %>%
  rownames_to_column("SampleID") %>%
  select(SampleID, transect_name)

sample_cols <- intersect(colnames(pathway_df), sample_meta$SampleID)
id_col <- colnames(pathway_df)[1]

pathway_long <- pathway_df %>%
  rename(Pathway = all_of(id_col)) %>%
  select(Pathway, all_of(sample_cols)) %>%
  pivot_longer(-Pathway, names_to = "SampleID", values_to = "Abundance") %>%
  left_join(sample_meta, by = "SampleID")

pathway_summary <- pathway_long %>%
  group_by(Pathway, transect_name) %>%
  summarize(mean_abundance = mean(Abundance), .groups = "drop") %>%
  pivot_wider(names_from = transect_name, values_from = mean_abundance, values_fill = 0) %>%
  mutate(log2FC = log2((Yungay + 1) / (Baquedano + 1)))

pathway_stats <- pathway_long %>%
  group_by(Pathway) %>%
  summarize(
    p_value = suppressWarnings(wilcox.test(Abundance ~ transect_name)$p.value),
    .groups = "drop"
  ) %>%
  mutate(FDR = p.adjust(p_value, method = "BH")) %>%
  left_join(pathway_summary, by = "Pathway") %>%
  arrange(FDR, desc(abs(log2FC)))

write_tsv(pathway_stats, file.path(ATACAMA_ROOT, "results", "picrust2_pathway_differential.tsv"))

cat("显著差异通路（FDR < 0.05）数:", sum(pathway_stats$FDR < 0.05, na.rm = TRUE), "\n")
print(
  pathway_stats %>%
    filter(FDR < 0.05) %>%
    select(Pathway, log2FC, FDR)
)

    📊 输出：
显著差异通路（FDR < 0.05）数: 1
   Pathway   log2FC        FDR
1 PWY-7254 -1.22514 0.01051538

    这意味着在 BH 校正之后，真正还能留下来的显著通路只有 PWY-7254 一条，而且方向是 Baquedano 富集（因为 log2FC = log2((Yungay + 1)/(Baquedano + 1)) 为负）。

    这和上一章 ANCOM-BC 只留下 1 个稳定差异属的感觉是一致的：这套真实环境数据没有“满屏显著”的戏剧性结果，稳定信号其实很少。

    top_pathways <- pathway_stats %>%
  filter(FDR < 0.05) %>%
  slice_head(n = 12) %>%
  mutate(Direction = ifelse(log2FC > 0, "Yungay", "Baquedano"))

p_pathways <- ggplot(
  top_pathways,
  aes(x = log2FC, y = reorder(Pathway, log2FC), fill = Direction)
) +
  geom_col(width = 0.72) +
  geom_vline(xintercept = 0, linewidth = 0.35, color = "grey30") +
  scale_fill_manual(values = c("Baquedano" = "#C66B2D", "Yungay" = "#2D7D7D")) +
  labs(x = "log2 fold change (Yungay / Baquedano)", y = NULL, fill = "Transect") +
  theme_songlab() +
  theme(legend.position = "top")

save_plot_dual(p_pathways, "ch08_pathway_barplot", width = 9, height = 6.5)

    图 2：差异通路条形图。 当前 FDR < 0.05 的只有 PWY-7254，方向偏向 Baquedano。这个图不“热闹”，但这正是现实数据的样子。

    这一章最该怎么写讨论

    正确的写法不是说：

    “PICRUSt2 证明了某条 transect 在某功能上更强。”

    而应该说：

    “PICRUSt2 提示 Baquedano 与 Yungay 在预测功能潜力上存在有限但可见的差异，其中 PWY-7254 在多重校正后仍保持显著；鉴于 NSTI 中位数为 0.282，这些结果应视为探索性发现，仍需宏基因组或代谢组数据进一步验证。”

    这类表述通常更能过审稿。

    本篇小结

    这一篇我们跑通了 Atacama 数据的 PICRUSt2 流程，并得到了三个关键结论：

    第一，流程本身是跑通的。 ASV 表、代表序列、KO/EC 预测和 MetaCyc 通路都成功生成。

    第二，NSTI 质量一般。 中位数 0.282 说明荒漠土壤样本和参考基因组之间距离并不近，所以功能预测只能谨慎使用。

    第三，稳定的显著功能信号很少。 多重校正后只剩 1 条通路 PWY-7254，方向偏向 Baquedano。

    这并不意味着 PICRUSt2 没价值。它仍然能给你一个功能层面的“候选列表”，帮助你决定下一步是否值得上更贵的组学。
