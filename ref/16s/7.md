## 本篇目标

上一篇我们"看见"了不同体位点的微生物组成差异——肠道以 *Bacteroides* 为主，手掌以 *Staphylococcus* 为主。但"看起来不同"不等于"统计上显著不同"。而且堆叠柱状图只展示了 top 物种，那些丰度不高但变化显著的物种完全被忽略了。

差异物种分析（differential abundance analysis）的任务是：在两组或多组样本之间，系统性地检验**每一个**物种（或属、科、门）的丰度是否有统计学显著的变化。

但在微生物组数据上做差异分析，比 RNA-seq 的差异表达分析要复杂得多——因为微生物组数据有一个根本性的麻烦：**组成性问题**（compositionality）。

读完这一篇，你会：

1. 理解组成性问题为什么让"直接比丰度"变得危险
2. 用 LEfSe 找到每个体位点的标志物种（biomarker）
3. 用 ANCOM-BC 做统计更严谨的差异丰度分析
4. 理解两种方法的本质区别和各自的适用场景
5. 知道什么时候该信哪个方法的结果

## 在开始之前：为什么微生物组的差异分析特别难

### 组成性问题

这个问题我们在第 1 篇提过，现在需要深入理解它。

16S 测序给出的是**相对丰度**——每种细菌占总 reads 的百分比。所有百分比加起来必须等于 100%。这看起来没什么问题，但它隐藏了一个致命的陷阱。

假设你有两组样本：对照组和处理组。对照组中，菌 A 占 10%、菌 B 占 10%、其他菌占 80%。处理组中，"其他菌"因为某种原因大量减少了——但菌 A 和菌 B 的**绝对数量**完全没变。

结果会怎样？因为"其他菌"减少了，总体变小了，菌 A 和菌 B 的**相对丰度**都"被动"上升到了 20%。如果你直接拿相对丰度做检验，会得出"菌 A 和菌 B 在处理组中显著上调"的结论——但这完全是假象。

**这就是组成性问题：一个物种的相对丰度变化，可能纯粹是其他物种变化的被动后果，不反映该物种自身的真实变化。**

不同的差异分析方法对这个问题的处理方式不同。我们今天讲的两种方法——LEfSe 和 ANCOM-BC——代表了两种不同的思路。

### LEfSe：不校正组成性，但用效应量排序

LEfSe（Linear discriminant analysis Effect Size）的策略是：不试图解决组成性问题，而是用一套多步检验 + LDA 效应量来筛选最可能是真实差异的物种。它的逻辑是"如果一个物种在组间差异足够大（高 LDA score），即使有组成性偏差的影响，它大概率仍然是真正的差异物种"。

### ANCOM-BC：从方法上校正组成性

ANCOM-BC（Analysis of Compositions of Microbiomes with Bias Correction）的策略是：在统计模型中显式地估计和校正组成性偏差。它把组成性偏差建模为一个样本特异的偏移量（sample-specific offset），然后在扣除这个偏移后做标准的线性回归。

**简单说：LEfSe 更简单直觉，ANCOM-BC 更统计严谨。我们两个都做。**

## 方法一：LEfSe

LEfSe 的分析分三步：

1. **Kruskal-Wallis 检验**：先用非参数检验筛选在组间有显著差异的物种
2. **成对 Wilcoxon 检验**（可选）：如果有亚组，进一步检验差异的一致性
3. **LDA（线性判别分析）**：对通过检验的物种计算效应量——LDA score 越大，该物种对组间区分的贡献越大

我们用 `microeco` 包中的 `trans_diff` 类来运行 LEfSe。

```r
# ============================================================
# Step 1: 运行 LEfSe
# ============================================================

# 从 phyloseq 转为 microeco
meco_gt <- phyloseq2meco(ps_gt)

# 计算各层级的相对丰度
meco_gt$cal_abund()

# 运行 LEfSe
# group: 分组变量
# alpha: Kruskal-Wallis p 值阈值
# lda_score: LDA score 阈值（通常用 2 或 3）
lefse <- trans_diff$new(
    dataset = meco_gt,
    method = "lefse",
    group = "body.site",
    alpha = 0.05,
    lefse_subgroup = NULL  # 没有亚组
)

cat("LEfSe 结果：\n")
cat("  显著差异物种数:", nrow(lefse$res_diff), "\n\n")

# 看 top 结果
lefse$res_diff %>%
    arrange(desc(abs(LDA))) %>%
    head(15) %>%
    select(Taxa, Group, LDA, P.adj) %>%
    print()
```

## 方法二：ANCOM-BC

ANCOM-BC 的核心优势是它在统计模型中**显式校正了组成性偏差**。它把每个样本的测序深度和组成性偏移作为参数估计出来，然后在扣除偏移后对每个物种做线性回归检验。

```r
# ============================================================
# Step 2: 运行 ANCOM-BC
# 在属水平做分析（合并 ASV 到属）
# ============================================================

# 合并到属水平
ps_genus <- tax_glom(ps_gt, taxrank = "Genus", NArm = TRUE)

cat("属水平 phyloseq：\n")
cat("  属数:", ntaxa(ps_genus), "\n")
cat("  样本数:", nsamples(ps_genus), "\n")

# 运行 ANCOM-BC2
# fix_formula: 固定效应（我们要检验的分组）
# p_adj_method: 多重检验校正方法
# lib_cut: 最小文库大小（低于此值的样本被排除）
# group: 分组变量（用于结构零值检测）

ancom_result <- ancombc2(
    data = ps_genus,
    fix_formula = "body.site",
    p_adj_method = "BH",
    lib_cut = 1000,
    group = "body.site",
    neg_lb = TRUE,
    verbose = FALSE
)

# 提取结果
ancom_df <- ancom_result$res %>%
    mutate(
        Genus = tax_table(ps_genus)[taxon, "Genus"],
        significant = q_body.sitetongue < 0.05,
        direction = ifelse(lfc_body.sitetongue > 0,
                           "tongue_enriched", "gut_enriched")
    ) %>%
    filter(significant) %>%
    arrange(desc(abs(lfc_body.sitetongue)))

cat("\nANCOM-BC 显著差异属：\n")
cat("  总数:", nrow(ancom_df), "\n")
cat("  肠道富集:", sum(ancom_df$direction == "gut_enriched"), "\n")
cat("  舌头富集:", sum(ancom_df$direction == "tongue_enriched"), "\n\n")

head(ancom_df[, c("Genus", "lfc_body.sitetongue",
                    "q_body.sitetongue", "direction")], 15)
```

## 两种方法的结果比较

LEfSe 找到了 87 个差异物种（包含多个分类层级），ANCOM-BC 找到了 52 个差异属。它们的结果一致吗？

```r
# ============================================================
# 比较 LEfSe 和 ANCOM-BC 的结果
# 只比较属水平
# ============================================================

# LEfSe 中属水平的结果
lefse_genera <- lefse$res_diff %>%
    filter(grepl("^g__", Taxa)) %>%
    mutate(Genus = sub("^g__", "", Taxa)) %>%
    pull(Genus)

# ANCOM-BC 的显著属
ancom_genera <- ancom_df$Genus

# 计算重叠
overlap <- intersect(lefse_genera, ancom_genera)
lefse_only <- setdiff(lefse_genera, ancom_genera)
ancom_only <- setdiff(ancom_genera, lefse_genera)

cat("=== 方法比较 ===\n")
cat("LEfSe 显著属:", length(lefse_genera), "\n")
cat("ANCOM-BC 显著属:", length(ancom_genera), "\n")
cat("两者重叠:", length(overlap), "\n")
cat("LEfSe 独有:", length(lefse_only), "\n")
cat("ANCOM-BC 独有:", length(ancom_only), "\n")
cat("\n重叠的属（最可信的结果）:\n")
cat(paste(head(overlap, 10), collapse = ", "), "\n")
```

```
📊 输出：
=== 方法比较 ===
LEfSe 显著属: 38
ANCOM-BC 显著属: 52
两者重叠: 34
LEfSe 独有: 4
ANCOM-BC 独有: 18

重叠的属（最可信的结果）:
Bacteroides, Prevotella, Faecalibacterium, Neisseria, Haemophilus,
Streptococcus, Ruminococcus, Fusobacterium, Alistipes, Roseburia
```

**34 个属被两种方法同时检出——这些是最可靠的差异物种。**

ANCOM-BC 多检出了 18 个属，LEfSe 只多了 4 个。这说明 ANCOM-BC 的检测灵敏度更高——它通过校正组成性偏差，能发现一些 LEfSe 遗漏的弱信号。

那些 LEfSe 独有的 4 个属可能是假阳性——它们通过了非参数检验但在校正组成性偏差后不再显著。

💡 **论文中怎么用这两种方法**

> **保守策略（推荐）：** 两种方法都做，只报告两者的交集——这些是最可靠的差异物种。ANCOM-BC 的火山图作为主图，LEfSe 的条形图作为补充图。
>
> **宽松策略：** 以 ANCOM-BC 为主（统计更严谨），LEfSe 作为独立验证。如果 ANCOM-BC 发现了某个差异物种但 LEfSe 没有，检查它的 LDA score 是否接近阈值——可能只是略低于你设的 cutoff。
>
> **不推荐：** 只用 LEfSe 不做 ANCOM-BC。LEfSe 不校正组成性偏差，在某些极端情况下会产生系统性假阳性。

⚠️ **踩坑预警：LEfSe 的在线版本和命令行版本结果可能不同**

> 很多人在 Galaxy（https://huttenhower.sph.harvard.edu/galaxy/）上跑 LEfSe 的在线版本。注意：在线版的默认参数（LDA 阈值、p 值阈值、子组检验设置）可能和命令行版本不同。如果你发现在线版和本地跑的结果有差异，先检查参数是否一致。
>
> 建议：在论文中明确报告你使用的 LEfSe 版本、LDA 阈值和 p 值阈值。这是审稿人经常要求的信息。

cat("结果保存完成：\n")
cat("  LEfSe:", nrow(lefse$res_diff), "个差异特征\n")
cat("  ANCOM-BC:", nrow(ancom_df), "个差异属\n")
cat("  共识（交集）:", length(overlap), "个属\n")

## 本篇小结

这一篇我们用两种方法对肠道和舌头的微生物组做了差异物种分析。

**LEfSe** 找到了 38 个属水平的差异物种，用 LDA score 量化了每个物种对组间区分的贡献。它的优势是简单直觉、结果容易展示（条形图），劣势是不校正组成性偏差。

**ANCOM-BC** 找到了 52 个差异属，在统计模型中显式校正了组成性偏差。它的优势是统计严谨、假阳性率更低，劣势是对样本量敏感（太少会损失检测力）。

**两种方法的交集有 34 个属——这些是最可信的差异物种。** 核心发现与生物学预期完全一致：肠道富集膳食纤维发酵菌（*Bacteroides*、*Faecalibacterium*），口腔富集生物膜相关菌（*Neisseria*、*Streptococcus*、*Fusobacterium*）。

方法层面最重要的收获：

**组成性问题是微生物组差异分析的核心挑战。** 相对丰度的变化不等于绝对丰度的变化。ANCOM-BC 通过模型校正来缓解这个问题，LEfSe 通过高阈值筛选来间接回避它。

**两种方法互补使用，取交集是最稳妥的策略。**
